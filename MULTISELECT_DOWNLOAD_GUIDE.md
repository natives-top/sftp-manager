# 多选文件下载功能说明

## 功能概述

现代化 SFTP 文件管理器现在支持**多选文件批量下载**功能，用户可以一次选择多个文件或文件夹并同时下载它们。

## 使用方法

### 1. 复选框选择
- **单个选择**: 点击文件行前的复选框
- **取消选择**: 再次点击复选框
- **全选**: 点击表头的复选框，选中所有文件
- **取消全选**: 再次点击表头复选框

### 2. 键盘快捷键
#### Ctrl/Cmd + Click (按住Ctrl或Cmd键点击)
- 切换单个文件的选择状态
- 可以建立不连续的多选

```
示例:
1. 点击文件 A (选中 A)
2. Ctrl+点击文件 C (选中 A 和 C, B 未选中)
3. Ctrl+点击文件 A (取消 A, 现在只选中 C)
```

#### Shift + Click (按住Shift键点击)
- 从上一个选中的文件到当前文件进行范围选择
- 自动填充中间的所有文件

```
示例:
1. 点击文件 A (选中 A)
2. Shift+点击文件 D (选中 A, B, C, D)
```

### 3. 工具栏按钮
在文件列表上方的工具栏中有以下按钮:

| 按钮 | 功能 |
|------|------|
| **Select All** | 选中所有文件 |
| **Deselect All** | 取消选中所有文件 |
| **Download (N)** | 下载已选中的N个文件(仅当有选中文件时显示) |

### 4. 右键菜单
- 当选中**多个文件**时,右键菜单显示 **Download (N)** 选项
- 当选中**单个文件**时,右键菜单显示 **Download** 选项

## 功能特性

✅ **灵活的选择方式**
- 复选框直观操作
- 支持 Ctrl/Shift 多选快捷键
- 支持全选/取消全选

✅ **实时反馈**
- 选中行高亮显示
- 工具栏实时显示选中数量
- 菜单项显示操作文件数

✅ **批量下载**
- 为每个文件创建独立的传输任务
- 多个文件可同时传输
- 每个文件有单独的进度条

✅ **与现有功能兼容**
- 支持文件和文件夹
- 不影响上传、收藏、删除等其他操作
- 双击仍然可以进入文件夹(在未按修饰键时)

## 工作流示例

### 场景1: 下载特定的几个文件
```
1. 在文件列表中, 点击第一个文件的复选框
2. 按住 Ctrl, 点击其他想要的文件的复选框
3. 点击工具栏的 "Download (3)" 按钮
4. 选择保存位置
5. 文件开始下载, 可在下方的传输任务面板中查看进度
```

### 场景2: 下载一个范围内的所有文件
```
1. 点击范围内的第一个文件
2. 按住 Shift, 点击范围内的最后一个文件
3. 所有中间的文件都被选中了
4. 在右键菜单中选择 "Download (5)"
5. 确认保存位置后, 所有文件同时开始下载
```

### 场景3: 快速下载整个目录
```
1. 点击表头的复选框执行全选
2. 点击工具栏的 "Download (全部数量)" 按钮
3. 整个目录的所有文件将被下载
```

## 技术实现

### 状态管理
```typescript
// 存储选中的文件索引集合
const [selectedFileIndices, setSelectedFileIndices] = useState<Set<number>>(new Set())

// 记录上次选中的文件索引,用于 Shift 范围选择
const [lastSelectedIndex, setLastSelectedIndex] = useState<number | null>(null)
```

### 选择处理
- **单击**: 仅选中该文件
- **Ctrl+单击**: 切换该文件的选择状态
- **Shift+单击**: 选中从上次选中到现在的文件范围

### 批量下载
- 遍历所有选中的文件
- 为每个文件创建独立的 TransferTask
- 调用现有的 `handleDownload()` 函数
- 完成后自动清除选择状态

## 性能考虑

- 选择状态使用 `Set<number>` 存储,O(1) 查询性能
- 批量下载不会阻塞 UI
- 每个文件的传输任务独立进行
- 可同时处理多个文件的传输

## 已知限制

- 网络速度限制了同时下载的效率(取决于 SFTP 服务器配置)
- 选择状态在导航到其他目录时会清除(这是预期行为)
- 单次操作的最大文件数受系统内存限制
